# Шаги отладки WebView

## Текущая ситуация
✅ Заголовок "CODELAB: Chat" отображается
❓ Остальные компоненты не видны

## Шаги для диагностики

### 1. Открыть Developer Tools

В окне с запущенным плагином:
1. `Help > Toggle Developer Tools`
2. Перейти на вкладку `Console`
3. Проверить наличие ошибок (красный текст)

### 2. Проверить Elements

1. В Developer Tools перейти на вкладку `Elements`
2. Найти `<div id="root">`
3. Проверить, какие компоненты отрендерились внутри

### 3. Проверить Network

1. Перейти на вкладку `Network`
2. Перезагрузить webview (`Cmd+R` или `Ctrl+R`)
3. Проверить, что все файлы загрузились:
   - `index.js` (зелёный статус 200)
   - `index.css` (зелёный статус 200)
   - `react-vendor-*.js` (зелёный статус 200)
   - `markdown-*.js` (зелёный статус 200)

### 4. Проверить стили

Возможно компоненты есть, но не видны из-за CSS. В Console выполнить:

```javascript
document.querySelector('#root').innerHTML
```

Это покажет, что реально отрендерилось.

## Возможные проблемы

### Проблема 1: CSS не загрузился

**Симптомы**: Компоненты есть в DOM, но не видны

**Решение**: Проверить в Network, что `index.css` загрузился

### Проблема 2: JavaScript ошибка

**Симптомы**: Ошибка в Console

**Решение**: Посмотреть stack trace ошибки

### Проблема 3: VS Code API не доступен

**Симптомы**: Ошибка "acquireVsCodeApi is not defined"

**Решение**: Проверить, что webview запущен в правильном контексте

## Временное решение для проверки

Добавьте в `webview/src/App.tsx` в начале компонента:

```typescript
useEffect(() => {
  console.log('App mounted!');
  console.log('Messages:', messages);
  console.log('IsLoading:', isLoading);
}, [messages, isLoading]);
```

Это поможет понять, монтируется ли компонент и какое у него состояние.
